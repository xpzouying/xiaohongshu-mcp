## 🔧 技术实现

### 核心模块

1. **album_manager.go** - 专辑管理模块
   - 获取专辑列表
   - 创建专辑
   - 添加笔记到专辑

2. **album_sync.go** - 同步服务模块
   - 加载分类结果
   - 批量同步
   - 错误处理和重试

3. **cmd/album-sync/main.go** - CLI 工具
   - 命令行参数解析
   - 进度显示
   - 报告生成

### API 端点

| 端点 | 方法 | 说明 |
|------|------|------|
| `/api/v1/albums/list` | GET | 获取专辑列表 |
| `/api/v1/albums/create` | POST | 创建专辑 |
| `/api/v1/albums/add_notes` | POST | 添加笔记到专辑 |

## ⚠️ 注意事项

### 1. API 限制

- 小红书 API 可能有频率限制
- 建议每批操作不超过 20 条笔记
- 操作间隔 2-3 秒

### 2. 认证要求

- 确保 `/tmp/cookies.json` 有效
- cookies 过期需要重新获取
- 建议使用最新登录的 cookies

### 3. 错误处理

- 自动重试失败的请求
- 80% 成功率即认为成功
- 详细错误信息记录在日志中

## 🔍 故障排查

### 问题 1: cookies 过期

**症状**: API 调用返回认证失败

**解决**:
```bash
# 重新获取 cookies
# 在浏览器中登录小红书，导出 cookies 到 /tmp/cookies.json
```

### 问题 2: 专辑创建失败

**症状**: 创建专辑 API 返回错误

**解决**:
- 检查专辑名称是否已存在
- 检查网络连接
- 查看详细错误日志

### 问题 3: 笔记添加失败

**症状**: 部分笔记无法添加到专辑

**解决**:
- 检查笔记 ID 是否有效
- 笔记可能已被删除
- 80% 成功率即认为成功
